# 生产落地待办清单

这份清单用于记录 AI-Interview 从 demo 项目走向可落地项目需要补齐的工作。

## P0 - 上线阻塞项

### 安全与密钥清理

- [x] 从工作区和 Git 追踪文件中移除真实密钥。
  - 验收标准：`backend/.env` 不再被 Git 追踪，仓库中只保留 `backend/.env.example`。
- [ ] 轮换所有已经暴露过的凭证。
  - 验收标准：DeepSeek API Key、SMTP 密码、JWT Secret、数据库密码以及其他泄露凭证全部废弃并替换。
- [ ] 清理 Git 历史中的敏感信息和运行时数据。
  - 验收标准：对完整 Git 历史执行密钥扫描，不再发现真实凭证。
- [x] 加入自动化密钥扫描。
  - 验收标准：CI 中集成 Gitleaks、TruffleHog 或同类工具，提交密钥时 CI 失败。
- [x] 清理已追踪的运行产物。
  - 验收标准：`node_modules`、`__pycache__`、`.pytest_cache`、`backend/chroma_data`、`backend/dump.rdb`、上传文件、IDE 配置等不再被 Git 追踪。

### 测试基础设施

- [x] 增加后端 pytest 测试基础。
  - 验收标准：`pytest` 可以在 CI 中运行，并使用隔离的测试配置、测试数据库或 mock 数据库层。
- [ ] 增加后端单元测试。
  - 验收标准：鉴权、配置、安全工具、文件校验、核心 service 等非 LLM 确定性逻辑有测试覆盖。
- [ ] 增加后端 API 集成测试。
  - 验收标准：登录注册、简历上传、面试生命周期、报告查询、管理员权限路径有测试覆盖。
- [ ] 增加前端测试基础。
  - 验收标准：引入 Vitest 和 Vue Test Utils，可以运行组件和 store 测试。
- [ ] 增加前端 E2E 冒烟测试。
  - 验收标准：Playwright 覆盖登录、简历上传、开始面试、SSE 展示、报告查看，并使用 mock 后端响应。
- [ ] 增加最小 CI 质量门禁。
  - 验收标准：后端测试、前端测试、类型检查、构建、lint、密钥扫描都在 PR 中执行。

### LLM 与智能体可靠性

- [x] 为 DeepSeek 调用增加明确的超时配置。
  - 验收标准：所有 LLM 调用都有连接、读取、总耗时超时，并能产生可识别的超时错误。
- [x] 为瞬时故障增加带退避的 retry。
  - 验收标准：429、5xx、连接重置、超时等场景有可测试的重试行为。
- [x] 为 DeepSeek 故障增加 circuit breaker。
  - 验收标准：连续失败后熔断，并返回用户可理解的降级响应。
- [x] 对 LLM 错误进行分类。
  - 验收标准：超时、限流、鉴权失败、token 超限、响应格式异常、服务商不可用映射到不同内部错误码。
- [ ] LLM 调用前增加 token 预算检查。
  - 验收标准：过长简历、历史对话、岗位信息会在超限前被截断或摘要化。
- [x] 为 ReAct 循环增加最大迭代次数。
  - 验收标准：智能体超过配置的工具/LLM 迭代次数后停止，并返回可恢复错误。
- [x] 校验工具调用参数。
  - 验收标准：非法工具调用会快速失败并返回结构化工具错误，不会无限循环。
- [x] 增加面试回合的兜底策略。
  - 验收标准：LLM 或工具失败时，用户看到明确的重试/降级提示，而不是只有“处理出错”。

### 生产配置

- [x] 移除生产启动时的 `Base.metadata.create_all`。
  - 验收标准：生产环境只通过 Alembic migration 管理数据库结构，应用启动不自动建表改表。
- [x] 增加启动时配置校验。
  - 验收标准：生产启动时如果必要环境变量、强密钥、CORS、上传路径、外部服务配置不合法，应用直接启动失败。
- [ ] 拆分开发、测试、预发、生产配置。
  - 验收标准：每个环境有明确配置，不能意外使用 demo 默认值。
- [x] 生产环境禁用 SQL echo 和 DEBUG 默认值。
  - 验收标准：生产日志不输出 SQL 明细或敏感 debug 信息。

### 业务逻辑迁移与旧逻辑清理

- [x] 创建面试接口改为使用 `job_id`，不再使用 `job_title` 查找或创建岗位。
  - 验收标准：前端创建面试时传 `resume_id + job_id`；后端只允许选择已存在且启用的岗位；找不到岗位时返回错误，不自动创建岗位。
- [x] 清理创建面试时的旧版 `question_count` 业务控制。
  - 验收标准：面试是否结束由智能体决策；`question_count`、`total_questions` 不再作为业务结束条件；如需保留限制，只保留 `max_turns`、`max_duration_minutes` 等安全上限。
- [x] 重新设计面试进度字段。
  - 验收标准：前端不再展示“第几题/共几题”这种固定题数进度，改为展示轮次、已回答数量、耗时、状态等更符合智能体流程的指标。
- [x] 移除创建面试时的预选题逻辑。
  - 验收标准：`POST /api/v1/interviews` 只创建会话，不调用 `search_questions`，不写入 `session.questions` 作为预设题池。
- [x] 将 RAG 检索收敛到智能体工具调用阶段。
  - 验收标准：题目检索发生在面试开始或每轮对话中，由智能体根据岗位、简历、历史回答实时调用检索工具。
- [x] 清理前端创建面试时传入的旧版 `config`。
  - 验收标准：前端不再硬编码 `{ question_count: 2, difficulty_distribution: ... }`；如需要配置，只传后端明确支持的新字段。
- [x] 梳理报告评分模型，清理 `overall_score=0` 的占位逻辑。
  - 验收标准：如果产品不需要数字总分，则移除或弱化 `overall_score`；如果需要数字总分，则定义可解释的评分来源和计算方式。
- [x] 重构报告中的优点/不足提取逻辑。
  - 验收标准：不再只靠“好/不错/准确/不足/缺/未”等关键词判断，改为基于画像、每轮评价或结构化总结。
- [x] 移除报告查询接口的重副作用。
  - 验收标准：`GET /reports/by-session/{session_id}` 只负责查询；报告缺失时不顺手修改 session 状态或同步生成报告，生成动作由明确的命令接口或后台任务处理。
- [x] 修正前端报告页命名。
  - 验收标准：如果路由参数传的是 `session_id`，前端变量和路由命名不再叫 `reportId`，避免误导。
- [x] 将报告生成迁移到后台任务。
  - 验收标准：面试结束后异步生成报告，前端展示生成中、生成成功、生成失败和重试状态。
- [x] 调整岗位删除策略。
  - 验收标准：岗位删除不破坏已有面试记录；优先使用软删除或禁用，必要时限制已被面试引用的岗位不能硬删除。
- [ ] 将题库同步 Chroma 改为后台任务。
  - 验收标准：题目新增、编辑、删除、启用/禁用后异步同步向量库，并有同步状态可查询。
- [ ] 让题库同步失败可见。
  - 验收标准：自动同步失败不会被静默吞掉，后台能看到失败原因并支持重试。
- [x] 明确普通用户是否允许访问题库接口。
  - 验收标准：若题库只供管理员和智能体使用，则普通用户不能直接列表/查看题目详情；若允许访问，则明确返回字段不泄露参考答案等内部信息。
- [ ] 后台权限从 `is_admin` 扩展为 RBAC。
  - 验收标准：岗位管理、题库管理、用户管理、数据看板等后台能力可以按角色分配权限。

## P1 - 产品稳定性与安全性

### 文件上传与个人信息保护

- [x] 停止将上传简历作为公开静态文件暴露。
  - 验收标准：访问简历文件必须经过鉴权，或使用签名 URL。
- [x] 按文件真实内容校验上传文件，而不只看 MIME type。
  - 验收标准：校验 PDF/DOCX magic bytes 和解析兼容性。
- [ ] 增加恶意文件扫描或隔离流程。
  - 验收标准：文件在解析或下载前经过扫描。
- [ ] 隔离文档解析过程。
  - 验收标准：PDF/DOCX 解析在 worker 或沙箱进程中执行，并设置超时和内存限制。
- [ ] 增加简历和面试记录的数据保留与删除规则。
  - 验收标准：用户删除数据时，DB、Redis、上传文件、报告、向量数据都能删除或匿名化。
- [ ] 日志脱敏。
  - 验收标准：邮箱、token、密钥、简历全文、原始面试回答默认不进入日志。

### 异步任务与状态管理

- [ ] 将长耗时任务从请求处理器中移出。
  - 验收标准：简历解析、报告生成、RAG 索引、重型 LLM 工作流通过队列执行。
- [ ] 增加任务状态跟踪。
  - 验收标准：前端可以展示 pending、running、failed、retrying、completed、canceled 等状态。
- [x] 明确面试状态机。
  - 验收标准：面试 session 的状态变化由状态机或等价校验保护。
- [x] 增加单场面试的并发控制。
  - 验收标准：同时 chat、skip、start、end 不会破坏会话状态，也不会重复写消息。
- [ ] 增加 SSE 断连处理。
  - 验收标准：用户断连不会留下不可恢复的半提交面试回合。

### API 契约与错误处理

- [x] 定义统一 API 响应和错误结构。
  - 验收标准：所有接口稳定返回 `code`、`message`、`request_id`，并可携带结构化 `details`。
- [ ] 用领域错误替代泛化异常处理。
  - 验收标准：预期失败不再靠宽泛的 `except Exception` 吞掉根因。
- [ ] 前端增加用户可理解的错误映射。
  - 验收标准：鉴权、上传、LLM 超时、报告生成失败、网络错误都有明确提示。
- [ ] 审查 OpenAPI 并生成前端 API 类型。
  - 验收标准：前端请求/响应类型由后端 schema 生成或校验。

### 认证与授权

- [x] 增加登录、注册、验证码发送限流。
  - 验收标准：按 IP 和账号/邮箱维度限流。
- [x] 增加登录失败锁定或渐进延迟。
  - 验收标准：连续失败登录会变慢或被暂时阻止。
- [x] 重构 refresh token 存储方式以适配规模化。
  - 验收标准：登出和 token 撤销不需要扫描所有 Redis key。
- [ ] 重新评估浏览器 token 存储方案。
  - 验收标准：明确记录 `localStorage` token 被 XSS 窃取的风险，或改为安全 HttpOnly Cookie 方案。
- [ ] 增加后台 RBAC 权限模型。
  - 验收标准：可配置超级管理员、内容管理员、客服/只读管理员等角色。
- [ ] 增加管理员审计日志。
  - 验收标准：用户状态修改、题目编辑、导入、报告访问都有审计记录。

### 可观测性

- [x] 增加结构化日志。
  - 验收标准：日志为 JSON 或其他机器可解析格式，并包含稳定字段。
- [x] 增加 request id 和 trace id。
  - 验收标准：每个 API 请求、SSE 流、LLM 调用、后台任务都能关联追踪。
- [ ] 增加指标监控。
  - 验收标准：暴露请求耗时、错误率、队列等待时间、LLM 耗时、token 用量、重试次数、RAG 命中率、报告生成耗时等指标。
- [x] 增加健康检查和就绪检查。
  - 验收标准：区分应用进程健康，以及 DB、Redis、Chroma、DeepSeek、SMTP、Embedding 服务是否就绪。
- [ ] 增加告警阈值。
  - 验收标准：服务商故障、队列堆积、高错误率、DB/Redis 异常会触发告警。

## P2 - 工程加固与可维护性

### 前端质量

- [ ] 增加 ESLint 和格式化。
  - 验收标准：CI 对 lint 和格式化错误失败。
- [ ] 增加路由守卫测试。
  - 验收标准：游客、普通用户、管理员、token 过期、非管理员访问后台等路径被覆盖。
- [ ] 加固 API 客户端的 retry/refresh 逻辑。
  - 验收标准：多个并发 401 不会触发多次 refresh 竞争。
- [ ] 增加 SSE 离线和重连状态。
  - 验收标准：面试房间可以稳定处理断网和恢复。
- [ ] 所有异步操作增加可见的加载和失败状态。
  - 验收标准：上传、解析、面试回合、报告生成、后台导入不会让界面卡死。

### 数据与数据库

- [ ] 审查 Alembic migration。
  - 验收标准：全新数据库只靠 migration 就能创建完整结构。
- [ ] 为高频查询增加索引。
  - 验收标准：鉴权查询、面试列表、后台看板、报告、题库搜索路径的索引经过审查。
- [ ] 增加备份与恢复流程。
  - 验收标准：在预发环境完成一次从备份恢复。
- [ ] 增加向量数据库迁移/重建策略。
  - 验收标准：Chroma 数据可以从主数据库题库记录重建或迁移。
- [ ] 审查数据库事务边界。
  - 验收标准：DB、Redis、LLM 混合流程不会留下不可恢复的半更新状态，或具备补偿机制。

### 依赖与发布管理

- [ ] 增加依赖漏洞扫描。
  - 验收标准：CI 报告 Python 和 npm 依赖漏洞。
- [ ] 固定并审查依赖版本。
  - 验收标准：依赖更新由 lockfile 和审查流程控制。
- [ ] 增加前后端 Dockerfile。
  - 验收标准：应用可以从可复现容器构建并运行。
- [ ] 增加预发环境部署。
  - 验收标准：预发环境使用接近生产的 DB、Redis、对象存储和密钥管理。
- [ ] 增加发布检查清单。
  - 验收标准：迁移、回滚、冒烟测试、监控、密钥轮换步骤都有文档。

### 文档

- [ ] 修复 README 编码。
  - 验收标准：中文内容以 UTF-8 正常显示。
- [ ] 增加本地开发指南。
  - 验收标准：新开发者能按文档启动后端、前端、DB、Redis、Chroma、Ollama。
- [ ] 增加环境变量说明。
  - 验收标准：每个环境变量都有用途、示例、必填/可选状态、生产建议。
- [ ] 增加故障处理手册。
  - 验收标准：DeepSeek 故障、Redis 故障、数据库迁移失败、密钥泄露都有处理流程。
- [ ] 增加隐私与数据处理文档。
  - 验收标准：明确简历、面试回答、评分、报告如何存储和删除。

## 建议里程碑

### 里程碑 1 - 先止血

- 密钥清理与轮换
- Git 历史清理
- 运行产物清理
- 基础 CI、密钥扫描和构建检查

### 里程碑 2 - 让改动可验证

- 后端 pytest 基础
- 前端 Vitest 基础
- API 集成测试
- CI 质量门禁

### 里程碑 3 - 让面试流程符合当前产品设计

- 创建面试改用 `job_id`
- 移除自动创建岗位
- 清理 `question_count` 和固定总题数进度
- 移除创建时预选题
- 将 RAG 检索放回智能体实时工具调用

### 里程碑 4 - 让智能体可靠

- LLM timeout/retry/circuit breaker
- ReAct 迭代上限
- LLM/工具 typed error
- SSE 恢复与清晰错误状态

### 里程碑 5 - 生产平台化

- Alembic-only 数据库结构管理
- 后台任务队列
- 上传安全
- 结构化日志、指标、trace、健康检查

### 里程碑 6 - 合规与运维

- PII 删除与保留策略
- 后台 RBAC 与审计日志
- 备份与恢复
- 预发环境和发布手册
